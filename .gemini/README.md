# Gemini CLI Project Notes

This document summarizes the work done and current state of your Gmail AI classification project with the Gemini CLI.

## Project Overview
This application categorizes your Gmail emails using a two-stage classification process:
1.  **Gemini API Classification:** Targets a specific number of emails (currently 2000) for initial high-quality classification.
2.  **Local ML Model Classification:** After Gemini completes its target, a local Machine Learning model classifies all remaining uncategorized emails.

## Setup Instructions

### 1. Database Setup (MySQL)
The project uses a MySQL database. Ensure your database is running. If you're using Docker Compose:
```bash
docker-compose up -d
```
You can view your database with Adminer at `http://localhost:8080` (User: `root`, Pass: `password`).

### 2. Environment Variables (.env)
Ensure your `.env` file in the project root is correctly configured with:
*   `GEMINI_API_KEY`: Your Google Gemini API key.
*   Gmail Authentication: Either `GMAIL_USER` and `GMAIL_APP_PASSWORD` (for IMAP) or `CLIENT_SECRET_PATH` pointing to your `client_secret.json` (for OAuth).

## How to Run the Classifier

To run the full two-stage classification process:
```bash
python main_pro.py --teach
```
This command will:
*   Check for already AI-classified emails.
*   Fetch only the necessary emails to reach the current Gemini target (2000).
*   Send these to the Gemini API in batches.
*   Update the `ai_category` and `ai_confidence` in your database.
*   Then, run the local ML model to classify all other uncategorized emails.

## Current Project State & Features Implemented

*   **Optimized Gemini Processing:** The application now intelligently targets a specific count (currently 2000) of emails for Gemini classification, preventing redundant API calls.
*   **Two-Stage Workflow:** Seamlessly transitions from Gemini API classification to a local ML model for comprehensive email categorization.
*   **Persistent Progress:** The system picks up where it left off; if a run is interrupted, it will fetch only the remaining emails needed for Gemini on the next execution.
*   **Database Indexing:** Performance-improving indexes have been added to the `emails` table on `manual_category` and `ai_category`.
*   **Robust Error Handling:** Gemini batch processing errors are caught, ensuring that only successfully processed emails are updated, allowing failed batches to be retried later.

## Next Steps / What You Can Do

*   **Adjust Gemini Target:** If you wish to change the target number of emails for Gemini classification, you will need to edit the `target_gemini_count` variable in `main_pro.py`.
*   **Train Local ML Model:** If you manually categorize some emails (via Adminer), you can then retrain your local ML model:
    ```bash
    python main_pro.py --train
    ```
*   **Run Local ML Prediction Only:** To run only the local ML prediction on uncategorized emails (without Gemini):
    ```bash
    python main_pro.py --predict
    ```
*   **Review and Refine:** Regularly check your database (via Adminer) and consider manually correcting any misclassifications. These manual corrections can then be used to retrain your local ML model for better accuracy.
